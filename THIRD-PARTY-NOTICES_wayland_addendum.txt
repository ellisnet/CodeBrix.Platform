THIRD-PARTY SOFTWARE NOTICES — WAYLAND HEAD ADDENDUM
(FORWARD-LOOKING, PRE-VETTED ALLOW-LIST AND GUARD-RAIL)

================================================================================
 WHAT THIS FILE IS  (READ FIRST)
================================================================================

This file is NOT a record of third-party code already present in this
repository — that is THIRD-PARTY-NOTICES.txt, in the same folder.

This file is a FORWARD-LOOKING, pre-vetted ALLOW-LIST for the upcoming native
Wayland head project (the planned "CodeBrix.Platform.UI.Runtime.Skia.Wayland"
project and its "CodeBrix.Platform.Runtime.Skia.Wayland.ApacheLicenseForever"
NuGet package). It exists for two reasons:

  1. GUARD-RAIL (while the work is in progress). During the Wayland head work
     we may PULL CODE FROM, VENDOR, GENERATE FROM, or even merely READ ONLY the
     sources enumerated in Parts B through E below. If a source is NOT listed
     here, we do not pull from it and we do not read it. When a genuinely new
     need arises, the source must first be added to this file (and license-
     vetted) BEFORE anyone touches it. "Vet first, then use."

  2. STAGING AREA (for when the work is done). Each source actually used will
     be migrated into THIRD-PARTY-NOTICES.txt (as a numbered item plus, where
     needed, its full license text in that file's Part D) once the Wayland head
     ships. See Part G.

Background and rationale live in:
  ~/ClaudeHome/REPORT_codebrix_native_wayland_head_feasibility_2026-07-01.txt

 THE HARD LICENSING BORDER  (NON-NEGOTIABLE)
================================================================================

IN-BOUNDS  (permissive; may enter the repo and the NuGet):
    MIT / Expat, Apache-2.0, BSD-2-Clause, BSD-3-Clause, zlib, ISC,
    X11, HPND ("permission ... without fee"), and public-domain / CC0.

OUT-OF-BOUNDS  (must never enter the repo or the NuGet):
    ANY copyleft — GPL, LGPL, MPL, EPL, EUPL, AGPL (weak or strong) —
    AND anything commercial / source-available / non-standard
    (e.g. BSL, SSPL, Elastic, "custom", "all rights reserved").

RUNTIME-STACK EXCEPTION:
    Runtime P/Invoke or dlopen of SYSTEM-PROVIDED libraries is allowed
    regardless of THEIR license, because we neither read, vendor, fork, nor
    redistribute their source. (Every Linux binary links glibc (LGPL); a
    Wayland client loads Mesa; libdecor may load a GTK or cairo plugin at
    runtime.) The border above governs INCORPORATED / DISTRIBUTED SOURCE only.

TARGET IS REAL, NATIVE WAYLAND — a genuine libwayland-client + xdg-shell wire-
protocol client. This project is NOT XWayland (the X11 compatibility server the
existing X11 head runs under on a Wayland desktop). No X11 code is in scope.

 PART B — VENDOR / FORK INTO THE TREE  (a ONE-AND-DONE import)
================================================================================

B1. NWayland  (the AvaloniaUI .NET Wayland protocol BINDINGS GENERATOR)
    Project URL:  https://github.com/AvaloniaUI/NWayland
    License:      MIT  (verified 2026-07-01: GitHub reports spdx_id "MIT")
    Scope:        We take ONLY the MIT bindings generator. We do NOT take,
                  read, or reference Avalonia's Wayland BACKEND (AGPL — see
                  Part F). NWayland is a separate, bindings-only, MIT repo.

    Use: fork / vendor a SNAPSHOT of the generator into the tree as a DEV-ONLY
    tool (e.g. under tools/ or a non-packed src/ project). Run it to emit C#
    proxy bindings which are COMMITTED to the Wayland head's Wayland_Bindings/
    folder (mirroring the committed-and-hand-written X11_Bindings/ precedent).

    ONE-AND-DONE requirements (so we never depend on Avalonia again):
      - Record the imported commit SHA + a verbatim copy of NWayland's MIT
        LICENSE alongside the vendored tool (the PORTING-NOTES pattern).
      - Absorb / inline any runtime support types the generated code needs, so
        that ZERO NWayland NuGet reference exists at build time or run time.
      - Thereafter, protocol updates (future Wayland versions) come from the
        upstream XML in Part C, run through OUR forked generator — never from a
        fresh Avalonia pull.

    Alternative generator, STUDY-ONLY (not the chosen path): WaylandSharp — see
    Part E.

================================================================================
 PART C — GENERATE-FROM  (protocol spec inputs; MIT; re-pulled from upstream)
================================================================================

The Wayland protocol definitions are DATA (XML) fed to our forked generator.
They come from the canonical upstream freedesktop.org projects — NOT from any
bindings library — so future protocol versions are always pulled from the
standard itself.

C1. wayland  (core protocol: wayland.xml)
    Project URL:  https://gitlab.freedesktop.org/wayland/wayland
    License:      MIT / Expat  (each protocol XML carries an MIT <copyright>)
    Interfaces of interest: wl_display, wl_registry, wl_compositor, wl_surface,
    wl_seat, wl_keyboard, wl_pointer, wl_touch, wl_output, wl_shm,
    wl_data_device_manager, wl_callback.

C2. wayland-protocols  (extension protocols)
    Project URL:  https://gitlab.freedesktop.org/wayland/wayland-protocols
    License:      MIT / Expat
    Protocols of interest: xdg-shell (wm_base/surface/toplevel/popup),
    xdg-decoration-unstable-v1, viewporter, fractional-scale-v1,
    cursor-shape-v1, text-input-v3 (IME; later), primary-selection (later).

================================================================================
 PART D — RUNTIME NATIVE LIBRARIES  (P/Invoke / dlopen; NOT vendored)
================================================================================

Bound to at run time via P/Invoke. Their source is never vendored or read, so
the runtime-stack exception applies; licenses are recorded here for provenance.
Licenses below were verified 2026-07-01 against the copies installed on the
development machine.

D1. libwayland-client / libwayland-cursor / libwayland-egl
    License: X11 (MIT-family permissive)   — Debian "License: X11".
    The core client library, cursor-theme helper, and the EGL window shim.

D2. libxkbcommon
    License: MIT / Expat.
    Keymap parsing from the compositor-supplied keymap FD. NOTE: we primarily
    REUSE the existing in-repo bindings (Part A, FrameBuffer head's
    xkbcommon_bindings.cs) rather than writing new ones.

D3. libdecor
    License: MIT / Expat  (Debian "License: Expat").
    Client-side window decorations (title bar, buttons, move/resize) on
    compositors that do not draw server-side decorations (notably GNOME).
    NOTE: libdecor may dlopen a GTK or cairo plugin at RUN TIME to theme the
    decorations; those plugins (LGPL / MPL) are OS-provided and are never read,
    vendored, or shipped by us — allowed under the runtime-stack exception.
    (A GPL-3.0+ build-helper script exists inside libdecor's own source tree;
    it is irrelevant because we bind only the MIT library and never vendor its
    source.)

D4. EGL / GLES  (Mesa)  — phase 2+/GPU path only
    License: MIT.
    For the later wl_egl_window + EGL/GLES -> SkiaSharp GRContext GPU path.
